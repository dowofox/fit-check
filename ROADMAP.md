# NAES Roadmap

## 작업 선택 규칙

항상 다음 순서로 작업한다.

1. P0
2. 가장 위 작업
3. 하나만 구현
4. 완료 후 다음 작업

절대로 여러 기능을 한 번에 구현하지 않는다.

새 작업을 시작하기 전에 이미 구현된 기능인지 먼저 확인한다.

## P0

- 링크 등록을 기본 등록 방식으로 변경
- 사진 등록 축소
- 추천 화면 단순화
- AI 내부 정보 숨김
- 소재 검증
- 프로필 단순화

## P1

- 추천 피드백
- 착용 기록
- 옷장 부족 분석
- 데이터 백업

## P2

- 서버 배포
- 로그인
- 동기화

## 완료 기준

- 기존 기능을 삭제하지 않는다.
- 사용자 화면은 결과 중심으로 단순해진다.
- `npm run lint`를 통과한다.
- `npx tsc --noEmit`을 통과한다.
- 로컬 커밋을 만들고 SHA를 보고한다.
- 사용자가 승인하기 전에는 `origin/main`에 push하지 않는다.
