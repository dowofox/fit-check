# NAES Roadmap

## 제품 기준

로드맵은 다음 두 가지 핵심 가치에 직접 기여하는 작업을 우선한다.

1. 내 옷으로 오늘 입을 코디를 추천한다.
2. 새 옷이 내 몸에 어떻게 맞는지 상품 실측을 기준으로 설명한다.

분석 항목을 늘리는 것보다 사용자가 더 좋은 추천을 받고, 결과를 이해하고, 자신의 데이터를 안전하게 관리하는지를 먼저 본다.

## 작업 선택 규칙

1. 긴급 P0가 있으면 가장 먼저 처리한다.
2. 긴급 P0가 없으면 아래 `다음 실행 순서`의 첫 미완료 작업을 선택한다.
3. 한 번에 하나의 작업만 구현한다.
4. 동일한 핵심 영역의 작업을 두 번 연속 선택하지 않는다.
5. 작업을 마치면 완료 상태와 `마지막 완료 영역`을 갱신한다.
6. 다음 작업이 핵심 가치에 직접 기여하는지 구현 전에 확인한다.
7. 단일 상품명·키워드 예외는 `CLASSIFICATION_ISSUES.md`에 기록하고 곧바로 제품 작업으로 승격하지 않는다.

### 긴급 P0 기준

다음 중 하나에 해당할 때만 진행 중인 순서를 중단하고 P0로 즉시 승격한다.

- 앱 크래시 또는 핵심 흐름 진입 불가
- 저장 데이터 손실·덮어쓰기·복구 불가 위험
- 사용자가 실제로 입거나 구매할 판단을 크게 그르칠 명백한 잘못된 추천

단일 상품의 분류 누락, 새 키워드 별칭, 낮은 영향의 표시 문제는 긴급 P0가 아니다. 반복성과 제품 영향을 확인한 뒤 해당 핵심 영역의 일반 작업으로 묶는다.

## 다음 실행 순서

긴급 P0가 새로 확인되지 않는 한 아래 순서를 유지한다.

1. [x] **추천 피드백** · 코디 추천
   - 사용자가 추천 코디에 `좋아요` 또는 `덜 추천` 피드백을 남길 수 있게 한다.
   - 피드백은 다음 추천 순위에 반영하되 내부 점수나 분석값은 화면에 노출하지 않는다.
   - 전체 아이템 조합별 피드백을 저장하며 같은 선택을 다시 누르면 해제할 수 있다.
2. [x] **오늘 입었어요** · 옷장 관리
   - 사용자가 실제로 입은 코디를 명시적으로 기록한다.
   - 코디 저장과 착용 기록을 구분하고, 사용자의 행동이 있을 때만 착용 이력을 갱신한다.
   - 같은 코디는 하루 한 번만 기록하며 저장만으로 착용 이력을 변경하지 않는다.
3. [x] **데이터 백업·복원** · 사용자 경험·데이터 안전성
   - 옷장, 프로필, 저장 코디, 피드백과 착용 기록을 내보내고 복원할 수 있게 한다.
   - 복원 전에 데이터 형식을 검증하고 기존 데이터를 덮어쓸지 명확히 확인한다.

마지막 완료 영역: **코디 추천**

## 핵심 영역별 미완료 작업

### 옷 등록

- [x] 등록 항목 ID 충돌 방지
  - 단일·일괄 등록 모두 시각과 임의값을 조합한 공통 ID 생성기를 사용한다.
  - 같은 밀리초에 등록이 겹쳐도 기존 항목 ID와 충돌해 저장 완료를 오인하지 않게 한다.

- [x] 상품 링크 내부 네트워크 접근 차단
  - 상품 URL과 모든 리다이렉트 목적지의 DNS 주소를 확인해 로컬·사설 네트워크 요청을 막는다.
  - 공개 쇼핑몰과 무신사 단축 링크는 유지하면서 안전하지 않은 주소는 서버 요청 전에 거부한다.

- [x] 등록 API 무한 대기 방지
  - 사진 분석과 상품 링크 추출 요청에 용도별 제한 시간을 적용한다.
  - 네트워크가 응답하지 않아도 로딩 상태가 끝나고 기존 실패 안내로 복구되게 한다.

- [x] 빌드 환경별 API 주소 설정
  - 소스 수정 없이 `EXPO_PUBLIC_API_BASE_URL`로 등록·분석 서버를 교체한다.
  - 환경변수가 없으면 기존 로컬 개발 서버를 사용하고 중복 슬래시를 방지한다.

- [x] 동시 옷장 변경 직렬화
  - 여러 등록과 상세 수정, 삭제, 착용 기록이 겹쳐도 하나의 원본을 순서대로 갱신한다.
  - 동시에 완료된 추가와 서로 다른 필드 수정이 마지막 쓰기에 의해 유실되지 않게 한다.

- [x] 미입력 사이즈 저장 정규화
  - 화면의 `사이즈 미입력` 안내 문구를 실제 `ClosetItem.size` 값으로 저장하지 않는다.
  - 기존 데이터의 안내 문구도 선택된 사이즈로 오해하지 않고 핏 추천 문구에서 제외한다.

- [x] 상품 추출 실패 안내 통일
  - 링크 등록과 상세 상품 확정에서 같은 서버 오류를 같은 사용자 행동으로 안내한다.
  - 잘못된 링크, 미지원 쇼핑몰, 연결 실패를 구분하고 직접 입력 fallback을 유지한다.
- [x] 상품 링크 요청 전 검증
  - 링크 등록과 상세 상품 확정에서 같은 URL 정규화·프로토콜 검증을 사용한다.
  - 잘못된 주소는 서버 요청 전에 안내하고, 스킴이 생략된 상품 도메인은 안전하게 보완한다.
- [x] 등록 결과 확인 동선 단순화
  - 링크·사진·직접 등록 후 사용자가 반드시 확인해야 하는 종류, 색상, 계절만 짧게 보여준다.
  - 공식 정보와 추정 정보의 차이는 내부 우선순위로 처리하고 화면에는 필요한 행동만 안내한다.
- [x] 지원 쇼핑몰과 추출 상태 안내 정리
  - 공식 상품 정보·소재·실측의 지원 범위를 사용자가 이해할 수 있는 결과 문구로 정리한다.
  - 개별 키워드 예외가 아니라 여러 상품에서 반복되는 추출 실패 유형을 기준으로 개선한다.

### 옷장 관리

- [x] 기준 옷 프로필 읽기 실패 시 삭제 차단
  - 옷 삭제 전에 기준 옷 연결을 확인할 프로필을 읽지 못하면 변경을 중단한다.
  - 일시적인 프로필 오류가 기준 옷과 옷장 사이의 연결을 깨뜨리지 않게 한다.

- [x] 옷 상세 읽기 실패와 삭제된 옷 구분
  - 옷장 저장소를 일시적으로 읽지 못하면 현재 옷을 찾을 수 없다는 안내를 표시하지 않는다.
  - 이미 열린 상세 정보는 보존하고 첫 진입과 갱신 실패 모두 같은 화면에서 다시 시도할 수 있게 한다.

- [x] 옷장 읽기 실패와 빈 목록 구분
  - 저장소를 일시적으로 읽지 못한 경우 기존 화면 목록을 빈 옷장으로 교체하지 않는다.
  - 저장된 데이터가 보존되었음을 안내하고 사용자가 같은 화면에서 다시 시도할 수 있게 한다.

- [x] 중복 옷 ID 저장 방지
  - 같은 등록 요청이 겹쳐도 동일한 `ClosetItem.id`는 한 번만 저장한다.
  - 상세 이동과 추천 조합 키가 중복 행 때문에 모호해지지 않게 저장 경계에서 차단한다.

- [x] 사용자 확인 정보 기반 옷장 검색 확장
  - 종류·색상·상품명뿐 아니라 계절, 사이즈, 핏과 공식 혼용률도 검색한다.
  - 상세 화면에서 본 값을 그대로 입력해 원하는 옷을 찾을 수 있게 한다.

- [x] 옷장 원본 읽기 실패 시 변경 차단
  - 조회 실패를 위한 빈 목록 fallback과 실제 변경 작업의 원본 읽기를 분리한다.
  - 원본 JSON이 깨졌거나 저장소 읽기에 실패하면 추가·수정·삭제를 중단해 기존 옷장을 보존한다.

- [x] 대표 이미지 로딩 fallback
  - 배경제거 이미지나 상품 이미지가 깨지면 다음 후보를 시도해 원본 사진까지 복구한다.
  - 모든 이미지 후보가 실패한 경우에만 기존 빈 이미지 안내를 표시한다.
- [x] 분류별 빈 상태 안내
  - 카테고리나 세부 분류에 옷이 없을 때 빈 그리드 대신 현재 조건을 설명한다.
  - 검색·정보 확인·보관 빈 상태와 구분해 다음 행동을 이해할 수 있게 한다.
- [x] 옷 삭제 실패 격리
  - 삭제 저장 실패와 마지막 옷을 정상 삭제한 빈 목록을 구분한다.
  - 실패 시 기존 옷장 목록을 유지하고 다시 시도할 수 있게 안내한다.
- [x] 옷장 이름·브랜드 검색
  - 세부 종류, 공식 상품명, 브랜드, 색상, 소재와 스타일 태그를 기존 필터 안에서 검색한다.
- [x] 옷장 등록일 정렬
  - 검색과 카테고리 필터 결과를 최근 등록순 또는 오래된순으로 확인한다.

- [x] 오늘 입었어요
  - `다음 실행 순서` 2번 작업과 동일하다.
- [x] 착용 이력 확인과 정정
  - 최근 입은 코디를 확인하고 잘못 기록한 이력을 취소할 수 있게 한다.
- [x] 옷 보관 상태 관리
  - 계절 보관이나 더 이상 입지 않는 옷을 삭제하지 않고 추천 대상에서 잠시 제외할 수 있게 한다.

### 코디 추천

- [x] 추천 피드백 읽기 실패와 미등록 구분
  - 피드백 저장소를 읽지 못하면 빈 선호로 추천 순서를 다시 계산하지 않는다.
  - 홈과 추천 화면 모두 마지막 추천을 유지하고 완전한 입력을 다시 불러오게 한다.

- [x] 추천 핵심 입력 부분 실패 차단
  - 옷장뿐 아니라 프로필이나 저장 코디 읽기가 실패해도 불완전한 입력으로 추천을 다시 계산하지 않는다.
  - 체형 보정과 저장 조합 제외가 정상인 마지막 추천을 유지하고 같은 화면에서 재시도한다.

- [x] 추천 입력 읽기 실패와 추천 부족 구분
  - 옷장 저장소를 일시적으로 읽지 못하면 빈 옷장 기준 추천을 계산하지 않는다.
  - 기존 추천을 보존하고 다시 시도할 수 있게 해 사용자가 옷을 다시 등록하도록 잘못 안내하지 않는다.

- [x] 상황 적합성과 품질 점수 분리
  - 데이트·데일리 같은 상황 키워드는 후보 선택과 정렬에만 사용한다.
  - 상황 태그 반복만으로 추천 점수와 등급이 올라가지 않게 품질 점수를 보존한다.

- [x] 홈 전용 날씨 추천 경계 복원
  - 홈과 홈에서 이어진 추천 더보기만 현재 날씨를 점수에 반영한다.
  - 코디 탭 일반 진입은 위치 요청 없이 현재 계절 기준으로 추천한다.

- [x] 공개 추천 API의 보관 옷 제외 정책 통일
  - 결과 객체 API와 배열 반환 API 모두 추천 후보 생성 전에 보관 중인 옷을 제외한다.
  - 호출 방식이 달라도 `추천에서 잠시 빼기` 설정이 항상 유지되도록 회귀 테스트한다.

- [x] 오래된 날씨 추천 캐시 만료
  - 날씨 원본 캐시와 홈 날씨 추천 캐시에 같은 2시간 유효시간을 적용한다.
  - 날씨가 없는 기본 추천 캐시는 revision이 같으면 계속 빠르게 복원한다.
- [x] 동점 추천 순서 결정성
  - 같은 옷장 데이터는 저장 배열 순서가 달라도 동일한 대표 추천과 대체 조합 순서를 만든다.
  - 기존 품질 우선순위가 모두 같을 때만 정렬된 아이템 ID 조합을 최종 기준으로 사용한다.
- [x] 대규모 옷장 후보 예산
  - 평소 규모의 결과는 유지하고 예상 조합이 5만 개를 넘을 때만 선호, 정보 완성도와 다양성을 기준으로 후보 풀을 제한한다.

- [x] 추천 피드백
  - `다음 실행 순서` 1번 작업과 동일하다.
- [x] 피드백 기반 추천 조정
  - 충분한 명시적 피드백이 있을 때 선호·비선호 경향을 추천 순위에 보수적으로 반영한다.
  - 특정 아이템 하나를 영구 제외하지 않고 사용자가 설정을 되돌릴 수 있게 한다.
- [x] 추천 결과의 실제 활용성 검증
  - 계절, 날씨, 저장 중복, 핵심 아이템 다양성이 실제 옷장 시나리오에서도 유지되는지 회귀 사례를 관리한다.

### 핏·사이즈 추천

- [x] 프로필 불완전 로드 덮어쓰기 차단
  - 프로필이나 기준 옷 목록을 읽지 못하면 빈 입력값과 빈 기준 옷을 저장하지 않는다.
  - 폼 값을 보존하고 읽기 완료 전 저장을 비활성화하며 같은 화면에서 다시 시도할 수 있게 한다.

- [x] 프로필 읽기 실패와 신체 정보 미등록 구분
  - 프로필 저장소를 일시적으로 읽지 못하면 빈 프로필 기준으로 사이즈를 계산하지 않는다.
  - 옷과 상품 정보는 유지하면서 프로필 재시도 후 실측 기반 추천을 복구한다.

- [x] 첫 수동 실측과 현재 상품 사이즈 연결
  - 현재 사이즈가 비어 있으면 처음 저장한 실측 행의 사이즈를 함께 선택한다.
  - 이미 선택한 사이즈가 있으면 다른 행을 추가해도 기존 선택을 유지한다.

- [x] 실측 행 식별과 숫자 범위 매칭 분리
  - `XL(33~34)`의 숫자 범위는 사용자 사이즈 참고에만 사용한다.
  - 범위 안 숫자 라벨의 별도 실측 행을 추가해도 기존 XL 행을 덮어쓰지 않는다.

- [x] 기준 옷 슬롯과 실제 카테고리 일치 검증
  - 기준 옷 ID가 남아 있어도 옷 종류가 바뀌었다면 이전 카테고리 슬롯에서 제거한다.
  - 상의·하의·아우터·신발 기준 실측이 서로 다른 카테고리 추천에 연결되지 않게 한다.

- [x] 신체 치수 자릿수 오입력 방지
  - 항목별로 넉넉한 cm 범위를 적용해 명백한 10배 입력 실수를 저장 전에 막는다.
  - 정상 범위의 소수 입력과 다양한 체형은 기존처럼 허용한다.
- [x] 중복 사이즈 실측 병합
  - 같은 상품의 동일 사이즈가 여러 행으로 나뉘어 추출되면 누락 실측을 상호 보완한다.
  - 값이 충돌할 때는 유효 필드가 더 완전한 행을 우선해 기존 판단을 유지한다.
- [x] 프로필 저장 실패 감지
  - 신체 치수와 기준 옷이 실제 저장된 경우에만 저장 완료를 안내한다.
  - 저장소 오류가 발생하면 입력값을 유지하고 다시 시도할 수 있게 한다.
- [x] 실측 기반 설명 품질 검증
  - 상의·아우터·하의의 대표 실측표로 추천 사이즈와 설명이 일관되는지 검증한다.
  - 라벨보다 상품별 실측, 사용자 신체 치수, 기준 옷 순으로 판단하는 원칙을 유지한다.
- [x] 부족한 실측 정보 안내 단순화
  - 추천을 계산할 수 없을 때 필요한 신체 치수나 상품 실측만 구체적으로 안내한다.
  - 근거가 부족한 경우 핏을 추측하지 않는다.
- [x] 기준 옷 비교 신뢰도 검증
  - 같은 카테고리와 유사한 형태의 기준 옷만 비교에 사용되는지 회귀 테스트를 보강한다.

### 사용자 경험·데이터 안전성

- [x] 홈 저장소 읽기 실패와 빈 대시보드 구분
  - 옷장, 프로필 또는 저장 코디를 읽지 못하면 0개와 추천 부족 상태로 바꾸지 않는다.
  - 마지막 대시보드를 유지하고 첫 로드 실패 시 미확인 표시와 같은 화면 재시도를 제공한다.

- [x] 저장 코디 읽기 실패와 빈 목록 구분
  - 저장 코디나 옷장 매칭 데이터를 일시적으로 읽지 못하면 기존 화면 목록을 교체하지 않는다.
  - 정상적인 미저장 상태와 구분해 데이터 보존 안내와 재시도 동작을 제공한다.

- [x] 착용 기록 원본 읽기 실패 시 변경 차단
  - 기록 조회 실패를 빈 착용 이력으로 오해해 새로 덮어쓰지 않는다.
  - 기록 추가·삭제 실패 시 과거 이력과 옷별 회전율 데이터를 함께 보존한다.

- [x] 추천 피드백 변경 유실 방지
  - 여러 코디의 피드백을 빠르게 바꿔도 변경을 순서대로 저장한다.
  - 기존 피드백 읽기에 실패하면 빈 목록으로 덮어쓰지 않고 변경을 중단한다.

- [x] 저장 코디 원본 읽기 실패 시 변경 차단
  - 조회 실패의 빈 목록 fallback을 저장·수정·삭제 원본으로 사용하지 않는다.
  - 저장소 읽기나 JSON 검증이 실패하면 기존 저장 코디를 그대로 보존한다.

- [x] 저장 코디 중복 저장 직렬화
  - 코디 저장·수정·삭제를 순서대로 처리해 빠른 연속 동작의 덮어쓰기를 막는다.
  - 동일 아이템 조합은 순서와 관계없이 저장 경계에서 한 번만 기록한다.
- [x] 불완전한 백업 생성 방지
  - 백업 원본은 한 번의 엄격한 저장소 읽기로 가져오고 읽기 실패를 빈 데이터로 대체하지 않는다.
  - 깨진 JSON이나 잘못된 루트 형식이 있으면 불완전한 백업 파일 생성을 중단한다.
- [x] 저장 코디 변경 실패 격리
  - 삭제·이름 수정 실패를 정상적인 빈 목록과 구분해 기존 화면 데이터를 유지한다.
  - 실제 저장이 완료된 경우에만 저장 코디 목록을 갱신한다.
- [x] 데이터 백업·복원
  - `다음 실행 순서` 3번 작업과 동일하다.
- [x] 홈 추천 영구 캐시와 날씨 중복 계산 제거
  - 추천 아이템 ID 중심의 영구 캐시를 복원하고 캐시 날씨와 실시간 날씨가 의미 있게 다를 때만 다시 계산한다.
  - 삭제된 옷이나 오래된 revision이 포함된 캐시는 안전하게 무효화한다.
- [x] 홈 포커스 재로드 최소화
  - 데이터 revision이 바뀌지 않은 홈 재진입에서는 전체 저장소 재로드를 생략하고 날씨만 갱신한다.
- [x] 추천 조합 공통 준비 계산 제거
  - 추천 결과를 바꾸지 않고 액세서리 옵션 생성, 핵심 코디 정렬과 저장 조합 비교의 반복 비용을 줄인다.
- [x] 저장 데이터 마이그레이션 검증
  - 새 optional 필드가 추가되어도 기존 옷장과 프로필이 깨지지 않는지 버전별로 확인한다.
- [x] 손상된 로컬 저장 데이터 격리
  - 배열이 아닌 저장값과 필수 필드가 깨진 항목을 읽기 단계에서 제외해 화면 크래시를 막는다.
- [x] 복구 가능한 오류 처리
  - 네트워크·분석 실패 시 입력한 정보가 사라지지 않고 다시 시도하거나 수동 등록으로 이어지게 한다.

## 분류 예외 처리

상품명, 소재명, 카테고리 별칭처럼 개별 분류 사례는 `CLASSIFICATION_ISSUES.md`에서 관리한다.

다음 조건을 충족하기 전에는 이 문서의 실행 큐에 넣지 않는다.

- 여러 상품 또는 쇼핑몰에서 반복된다.
- 공통 규칙으로 해결할 수 있다.
- 코디 추천이나 실측 기반 핏 설명에 의미 있는 영향을 준다.

## 완료 이력

기존 `P0 N차` 완료 목록은 Git 이력에 보존한다. ROADMAP은 완료 작업의 장기 보관소가 아니라 현재 제품 우선순위와 미완료 작업을 관리하는 문서로 사용한다.

## 완료 기준

- 기존 기능과 저장 데이터 호환성을 유지한다.
- 사용자 화면은 내부 분석보다 결과와 다음 행동을 중심으로 보여준다.
- 관련 회귀 테스트를 실행한다.
- `npm run lint`를 통과한다.
- `npx tsc --noEmit`을 통과한다.
- 검증이 모두 통과하면 로컬 커밋을 만들고 정해진 Git 절차에 따라 반영한다.
