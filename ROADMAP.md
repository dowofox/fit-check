# NAES Roadmap

## 작업 선택 규칙

항상 다음 순서로 작업한다.

1. P0
2. 가장 위 작업
3. 하나만 구현
4. 완료 후 다음 작업

절대로 여러 기능을 한 번에 구현하지 않는다.

새 작업을 시작하기 전에 이미 구현된 기능인지 먼저 확인한다.

## P0

현재 P0는 55차 작업을 진행한다.

- [x] 공식 소재의 재생섬유 혼용률 추출 지원
  - `텐셀`·`리오셀`·`lyocell`·`tencel`을 `리오셀`로 정규화한다.
  - `모달`·`modal`, `큐프로`·`cupro`를 공식 소재 항목에 보존한다.
  - 기존 면·폴리에스터 등과 섞인 혼용률에서 일부 소재만 누락되지 않게 한다.

완료된 P0 54차 항목:

- [x] 계절 판정의 보온 키워드 부분 문자열 오인 방지
  - `다운타운`을 다운 충전재로, `기모노`를 기모 소재로 판단하지 않는다.
  - `구스 다운`, `덕 다운`, `다운 베스트`, `기모 팬츠`는 기존처럼 겨울 근거로 유지한다.
  - 사진 분석 후처리와 공식 상품 링크 보정에 같은 예외를 적용한다.

완료된 P0 53차 항목:

- [x] 영문 확장 사이즈 표기 정규화
  - `EXTRA SMALL`, `EXTRA LARGE`, `DOUBLE EXTRA LARGE`, `TRIPLE EXTRA LARGE`를 각각 `XS`, `XL`, `XXL`, `XXXL`로 비교한다.
  - 상품 링크 실측 추출과 앱의 선택 사이즈 비교가 같은 결과를 사용한다.
  - 기존 숫자·FREE·복합 사이즈 표기는 유지한다.

완료된 P0 52차 항목:

- [x] 상품명 한 글자 소재의 부분 문자열 오인 방지
  - `울트라`, `모던`에 포함된 글자를 울 소재로 저장하지 않는다.
  - 독립된 `울`, `모`, `울 100%`, `모 100%`는 기존처럼 공식 소재 근거로 인정한다.
  - 카테고리·세부 종류·스타일 태그의 기존 테이블 분류는 유지한다.

완료된 P0 51차 항목:

- [x] 한 글자 울 소재의 부분 문자열 오인 방지
  - `울트라`처럼 다른 단어에 포함된 `울`을 방한 소재 근거로 사용하지 않는다.
  - 독립된 `울`, `울 니트`, `울 100%` 표기는 기존처럼 가을·겨울 근거로 인정한다.
  - 사진 분석과 공식 상품 링크의 계절 규칙에 같은 단어 경계 정책을 적용한다.

완료된 P0 50차 항목:

- [x] 사진 설명의 한 글자 린넨 오인 방지
  - `마감`, `마이크로화이버`에 포함된 한 글자 `마`를 린넨 계절 근거로 사용하지 않는다.
  - 사진 분석에서는 `린넨`과 `linen`처럼 명확한 소재 표현만 봄·여름 근거로 인정한다.
  - 공식 상품 링크의 구조화 소재와 기존 구체 품목 계절 규칙은 유지한다.

완료된 P0 49차 항목:

- [x] 모호한 슬링백·벨트백 상품명의 세부 분류
  - 신발 형태 단어가 함께 있는 슬링백은 가방이 아니라 신발로 분류한다.
  - 벨트 백, 웨이스트백, 힙색은 벨트가 아니라 가방으로 분류한다.
  - 단독 슬링백과 단독 벨트의 기존 분류는 유지한다.

완료된 P0 48차 항목:

- [x] 경량 추천 입력의 현재 사이즈 실측 보존
  - 홈과 추천 화면의 공통 경량 입력에 현재 선택 사이즈와 일치하는 실측 행 1개만 포함한다.
  - 전체 사이즈표는 추천 계산에서 제외해 초기 성능과 입력 일관성을 유지한다.
  - 선택 사이즈가 없거나 일치 행이 없으면 임의 행을 사용하지 않고 기존 보수적 fallback을 유지한다.

완료된 P0 47차 항목:

- [x] 공식 카테고리와 충돌하는 상품명 규칙 차단
  - 공식 상품 카테고리가 명확하면 같은 카테고리 그룹의 세부 분류 규칙만 적용한다.
  - 슬링백 슈즈를 가방으로, 벨트 백을 벨트로 오인하는 교차 카테고리 충돌을 막는다.
  - 공식 카테고리가 없거나 모호한 상품은 기존 상품명 기반 전체 규칙을 유지한다.

완료된 P0 46차 항목:

- [x] 의도한 착용감과 다른 실측 핏 경고 연결
  - 작음뿐 아니라 원하는 오버핏보다 작거나 딱 맞는 의도보다 큰 실측 결과도 코디 경고에 반영한다.
  - 신발과 액세서리는 자동 사이즈 적합도 경고 대상에서 제외한다.
  - 핏 경고는 추천 제외가 아니라 기존 점수·경고 정책 안에서만 보수적으로 반영한다.

완료된 P0 45차 항목:

- [x] 영문 장문 사이즈 라벨의 표준 매칭
  - `SMALL`, `MEDIUM`, `LARGE`, `X-LARGE`, `XX-LARGE`를 표준 문자 사이즈로 해석한다.
  - 서버 자동 추출과 앱의 직접 실측 입력이 같은 내부 라벨을 사용한다.
  - 쇼핑몰 원문 표기와 상품별 숫자 범위는 표시용 데이터에 그대로 보존한다.

완료된 P0 44차 항목:

- [x] 숫자 범위가 붙은 FREE 사이즈 라벨 지원
  - `ONE SIZE (44~66)`, `O/S(44-66)`처럼 참고 범위가 붙은 프리사이즈도 `FREE` 행으로 인식한다.
  - 상품별 숫자 범위는 해당 실측 행의 `numericRange`에만 보존하고 전역 사이즈 기준으로 사용하지 않는다.
  - 범위가 없는 FREE 별칭과 일반 복합 사이즈의 기존 동작은 유지한다.

완료된 P0 43차 항목:

- [x] 쇼핑몰 FREE 사이즈 별칭의 일관된 매칭
  - `FREE SIZE`, `ONE-SIZE`, `O/S` 같은 공식 표기를 서버와 앱에서 모두 `FREE`로 해석한다.
  - 원문 표기는 보존하되 상품 실측 행과 선택 사이즈 비교에는 공통 라벨을 사용한다.
  - 일반 문자·숫자·복합 사이즈의 기존 추출과 추천 동작은 유지한다.

완료된 P0 42차 항목:

- [x] 기준 옷 기반 사이즈 추천의 상충 안내 제거
  - 기준 옷 실측으로 추천 사이즈를 계산한 경우 신체 치수 부족 차단 카드를 함께 노출하지 않는다.
  - 신체 치수 기반 현재 사이즈 적합도와 기준 옷 기반 추천 사이즈 계산은 기존 방식대로 유지한다.
  - 기준 옷이 없거나 추천을 계산하지 못한 경우에는 기존 보완 안내를 계속 제공한다.

완료된 P0 41차 항목:

- [x] ProductGroup 변형 소재의 합의 기반 추출
  - 모든 변형의 공식 소재가 같을 때만 그룹 공통 소재로 사용한다.
  - 변형별 소재가 다르면 첫 변형을 임의 선택하지 않고 소재를 비운다.
  - 그룹 자체 소재와 기존 단일 Product 소재는 계속 최우선으로 사용한다.

완료된 P0 40차 항목:

- [x] ProductGroup 링크의 임의 변형 선택 방지
  - `ProductGroup`의 공통 상품 정보를 본상품 구조화 데이터로 인식한다.
  - 그룹 내부 첫 변형의 색상·이미지를 현재 선택값처럼 임의 저장하지 않는다.
  - 페이지가 제공하는 현재 변형 메타와 기존 단일 `Product` 추출은 유지한다.

완료된 P0 39차 항목:

- [x] 단일 구조화 소재 객체의 비율 보존
  - `{ name, percentage }` 형태의 공식 소재도 배열 항목과 동일하게 읽는다.
  - 소재명뿐 아니라 제공된 공식 혼용률을 함께 저장한다.
  - 문자열·배열·상품정보고시 소재 추출의 기존 동작은 유지한다.

완료된 P0 38차 항목:

- [x] 겉감·안감 공식 소재 구역 보존
  - 겉감과 안감이 각각 100%인 정상 혼용률을 합계 초과 오류로 버리지 않는다.
  - 여러 소재 구역에서 확인된 공식 소재와 비율을 모두 저장하고 표시한다.
  - 소재 정보가 없는 상품의 기존 partial 처리와 추천 fallback은 유지한다.

완료된 P0 37차 항목:

- [x] 공식 소재 별칭의 부분 문자열 오인 방지
  - `마이크로화이버` 안의 `마`를 린넨으로 오인하지 않는다.
  - `polyamide`를 `poly` 약칭 때문에 폴리에스터로 오인하지 않고 나일론 계열로 해석한다.
  - 독립된 `마`, `모`, `POLY` 표기의 기존 지원은 유지한다.

완료된 P0 36차 항목:

- [x] 사계절 마케팅 문구보다 구체 품목 우선
  - 패딩·다운·반팔·샌들처럼 물성이 분명한 품목은 올시즌 문구보다 먼저 판단한다.
  - 일반 데님처럼 구체 계절 단서가 없는 상품의 사계절 명시는 계속 인정한다.
  - 사진 등록과 링크 등록의 계절 우선순위를 같은 방향으로 유지한다.

완료된 P0 35차 항목:

- [x] 공식 상품 소매 형태의 계절 우선순위 정렬
  - 반팔·민소매·쇼츠 같은 구체적인 형태를 일반 보온 소재보다 먼저 적용한다.
  - 같은 상품명이 사진 등록과 링크 등록에서 서로 다른 계절로 저장되지 않게 한다.
  - 패딩·다운처럼 명확한 방한 품목의 기존 최우선 규칙은 유지한다.

완료된 P0 34차 항목:

- [x] 소량 혼방 소재의 계절 과잉 추론 방지
  - 공식 혼용률에서 비중이 작은 보조 소재 하나로 계절을 확정하지 않는다.
  - 상품명에 명시된 계절 소재는 기존처럼 강한 근거로 유지한다.
  - 구조화 혼용률이 없는 소재 요약은 기존 fallback을 유지한다.

완료된 P0 33차 항목:

- [x] 구조화 브랜드 배열과 공동 브랜드 보존
  - JSON-LD의 브랜드가 배열이어도 공식 브랜드명을 누락하지 않는다.
  - 공동 브랜드는 첫 항목을 임의 선택하지 않고 중복 제거한 이름을 함께 보존한다.
  - 기존 문자열·단일 객체 브랜드와 메타 fallback 동작은 유지한다.

완료된 P0 32차 항목:

- [x] 연관 상품 소재의 fallback 오인 방지
  - 본상품 소재가 구조화 데이터에 없을 때 연관·추천 상품 JSON의 소재를 대신 저장하지 않는다.
  - 상품정보고시와 본문 소재 fallback이 연관 상품 데이터보다 우선될 수 있게 탐색 범위를 정리한다.
  - 일반 JSON과 HTML 소재 추출의 기존 지원 범위는 유지한다.

완료된 P0 31차 항목:

- [x] 선택된 본상품의 공식 소재 우선 추출
  - URL 기준으로 선택된 본상품의 구조화 소재를 연관 상품보다 먼저 사용한다.
  - 무신사 공식 소재 API가 없을 때 본상품 구조화 소재, 페이지 전체 fallback 순으로 확인한다.
  - 기존 상품정보고시와 일반 HTML 소재 추출은 fallback으로 유지한다.

완료된 P0 30차 항목:

- [x] 추적 파라미터가 있는 본상품 URL 매칭
  - 공유 링크의 쿼리 파라미터가 달라도 origin과 pathname이 같으면 같은 상품으로 판단한다.
  - 정확히 같은 URL은 기존처럼 가장 높은 우선순위를 유지한다.
  - 경로가 다른 연관 상품은 본상품으로 선택하지 않는다.

완료된 P0 29차 항목:

- [x] 계층형 공식 상품 카테고리 보존
  - 구조화 카테고리 배열의 첫 값만 사용하지 않고 유효한 전체 경로를 보존한다.
  - `Apparel > Bottoms > Pants`처럼 구체 항목까지 분류 엔진에 전달한다.
  - 단일 문자열 카테고리의 기존 동작은 유지한다.

완료된 P0 28차 항목:

- [x] 다중 색상 상품의 현재 변형 오인 방지
  - 구조화 데이터가 전체 색상 옵션 배열이면 첫 색상을 현재 상품 색상으로 확정하지 않는다.
  - 선택 변형용 색상 메타가 있으면 그 값을 우선 사용한다.
  - 선택 색상 근거가 없으면 공식 색상을 비워 사진 분석 또는 사용자 확인으로 남긴다.

완료된 P0 27차 항목:

- [x] 직접 상품 변경 시 이전 공식 메타데이터 제거
  - 새 상품을 직접 입력할 때 이전 상품의 이미지, 소재, 색상, 카테고리를 넘겨받지 않는다.
  - URL 자동 추출 실패 후 직접 입력으로 전환하면 사용자가 붙여넣은 URL만 유지한다.
  - 기존 확정 상품의 단순 정보 수정은 현재 메타데이터를 그대로 유지한다.

완료된 P0 26차 항목:

  - 0 이하이거나 숫자가 아닌 실측값이 저장되지 않게 하고, 적어도 한 개의 유효 실측이 있을 때만 행을 저장한다.
  - 같은 사이즈를 다시 입력하면 중복 행을 만들지 않고 기존 행을 안전하게 교체한다.
- [x] 사이즈 추천 차단 상태의 다음 행동 안내 통일
  - 프로필 치수 부족, 상품 실측 부족, 상품 사이즈 미선택 상태를 구분해 사용자가 바로 보완할 수 있게 한다.
  - 자동 추천 대상이 아닌 신발과 액세서리에는 불필요한 차단 카드를 만들지 않는다.

완료된 P0 6차 항목:

- [x] 옷 상세 수정의 기본 정보 검증 연결
  - 상세 수정에서도 종류, 색상, 계절이 빈 값으로 저장되지 않게 등록과 같은 검증 규칙을 사용한다.
  - 기존의 불확실한 값은 사용자가 확인한 뒤 유지할 수 있게 해 수정 흐름을 막지 않는다.
- [x] 사진 없는 옷의 사진 보완 흐름 추가
  - 직접 등록한 옷 상세에서 사진을 선택해 대표 이미지를 보완할 수 있게 한다.
  - 기존 상품 이미지나 배경제거 이미지가 있으면 현재 이미지 우선순위를 유지한다.
- [x] 추천에 쓰기 어려운 옷의 보완 안내
  - 종류나 색상이 불확실해 추천에서 제외되거나 낮게 반영되는 옷을 옷장과 상세에서 구분한다.
  - 사용자가 바로 수정 화면으로 들어가 필요한 정보만 보완하게 한다.

완료된 P0 5차 항목:

  - 빈 이미지 주소를 이미지 컴포넌트에 전달하지 않고 옷장, 상세, 홈과 추천 카드에서 일관된 대체 화면을 보여준다.
  - 상품 링크와 사진으로 등록한 기존 이미지 우선순위는 유지한다.
- [x] 등록 경로별 필수 데이터 검증 통일
  - 링크, 사진, 직접 등록 모두 추천에 필요한 종류, 색상, 계절이 비어 있는 채 저장되지 않게 한다.
  - 자동 분석값이 불확실하면 저장을 막기보다 사용자가 확인할 항목을 짧게 안내한다.
- [x] 등록 출처와 무관한 추천 입력 정규화
  - 링크, 사진, 직접 등록으로 만든 옷이 같은 카테고리, 계절, 스타일 fallback 규칙을 거쳐 추천에 들어가게 한다.
  - 기존 저장 데이터의 빈 값도 추천 계산 전에 안전하게 정규화한다.

완료된 P0 4차 항목:

  - 저장 코디와 다른 버전에서도 내부 점수나 등급 대신 코디 이름과 구성을 보여준다.

  - 분석 기록에서도 점수 대신 날짜, 요약, 핵심 개선점을 빠르게 확인하게 한다.

  - 지원 상품에서는 자동 추출하고 실패해도 상품 등록은 유지하며 직접 입력으로 연결한다.
- [x] 상품 링크 등록과 이미지 AI 분석 장애 분리
  - 공식 상품 정보를 가져온 뒤 이미지 분석이 실패해도 확인된 상품 정보로 등록을 계속할 수 있게 한다.
  - 불확실한 종류, 색상, 계절은 사용자가 저장 전 또는 상세 화면에서 바로잡게 한다.
- [x] 최소 직접 등록 흐름 추가
  - 링크와 사진을 사용할 수 없을 때 종류, 색상, 계절, 사이즈만 입력해 옷장에 추가할 수 있게 한다.
  - 직접 등록은 상품 링크와 사진 등록을 대체하지 않는 마지막 보조 흐름으로 유지한다.
- [x] 등록부터 추천까지 핵심 흐름 통합 회귀 테스트 추가
  - 상품 링크 추출, 옷장 저장, 상세 수정, 추천 후보 반영의 연결이 끊기지 않는지 자동 검증한다.

- [x] 상품 링크 등록 지원 범위 점검

  - 지원 범위를 벗어난 링크는 성공처럼 보이지 않게 처리한다.

## 완료 기준

- 기존 기능을 삭제하지 않는다.
- 사용자 화면은 결과 중심으로 단순해진다.
- `npm run lint`를 통과한다.
- `npx tsc --noEmit`을 통과한다.
- 로컬 커밋을 만들고 SHA를 보고한다.
- 검증이 모두 통과하면 `origin/main`에 자동 push한다.
- 문제가 있거나 작업 범위가 모호하면 push하지 않고 사용자에게 먼저 묻는다.
