# NAES Roadmap

## 작업 선택 규칙

항상 다음 순서로 작업한다.

1. P0
2. 가장 위 작업
3. 하나만 구현
4. 완료 후 다음 작업

절대로 여러 기능을 한 번에 구현하지 않는다.

새 작업을 시작하기 전에 이미 구현된 기능인지 먼저 확인한다.

## P0

현재 P0는 15차 작업을 진행한다.

- [x] 공식 실측의 단면·둘레 기준 정규화
  - 허리·가슴·엉덩이·허벅지의 `둘레` 값은 단면 기준으로 변환하고 `단면` 값은 그대로 유지한다.
  - 기준이 명시되지 않은 값은 임의 변환하지 않아 공식 데이터 왜곡을 피한다.
- [ ] 상품 실측 직접 입력의 단면 기준 안내
  - 상의·하의 직접 입력에서 가슴, 허리, 엉덩이, 허벅지가 단면 기준임을 라벨과 설명으로 명확히 한다.
  - 저장 구조와 기존 실측 데이터는 변경하지 않는다.
- [ ] 단면·둘레 추출과 사이즈 추천 회귀 검증
  - 같은 옷의 허리단면 41cm와 허리둘레 82cm가 동일한 추천 입력값으로 정규화되는지 자동 테스트한다.
  - 명시적 둘레가 아닌 일반 `waist` 값은 기존처럼 유지되는지 확인한다.

완료된 P0 14차 항목:

- [x] 하의 품목별 계절 단서 확장
  - 버뮤다·쿨링 팬츠와 기모·코듀로이·울 팬츠를 같은 하의로 뭉뚱그리지 않고 계절 범위를 구분한다.
  - 사진 분석과 공식 상품명 보정이 같은 품목 단서를 사용하도록 맞춘다.
- [x] 신발·액세서리 계절 단서 확장
  - 샌들·방한 부츠·비니·머플러·밀짚모자처럼 계절성이 명확한 품목만 보수적으로 확정한다.
  - 일반 스니커즈, 가방, 주얼리는 근거 없이 사계절로 단정하지 않는다.
- [x] 사진 분석과 공식 상품 계절 규칙의 일치 회귀 검증
  - 같은 품목명이 사진 분석과 링크 등록에서 서로 다른 계절로 저장되지 않도록 대표 품목을 자동 테스트한다.
  - 더 구체적인 품목 단서가 일반 소재 단서보다 먼저 적용되는지 확인한다.

완료된 P0 13차 항목:

- [x] 홈 추천 빈 상태를 원인별 다음 행동으로 연결
  - 상의·하의 부족, 품질 기준 미달, 저장 조합 소진을 구분해 사용자가 이유를 바로 이해하게 한다.
  - 낮은 품질의 조합을 대신 노출하지 않고 기존 옷 추가 흐름으로 연결한다.
- [x] 계절 확인 필요 옷이 추천에 포함될 때 보완 동선 제공
  - 불확실한 계절을 중립적으로 허용한 코디는 결과에서 계절 확인이 필요하다는 점과 해당 옷을 알려준다.
  - 내부 신뢰도는 노출하지 않고 옷 상세 수정으로 바로 연결한다.
- [x] 홈과 추천 화면의 빈 상태 일치 회귀 검증
  - 같은 옷장, 저장 코디, 날씨 입력이면 두 화면이 같은 빈 상태 원인을 사용하도록 자동 테스트로 고정한다.
  - 날씨 갱신 뒤에도 품질 기준 미달 조합이 홈에 복원되지 않는지 확인한다.

완료된 P0 12차 항목:

- [x] 제안된 계절을 그대로 확인하는 동작
  - AI가 제안한 계절이 맞을 때 칩을 억지로 바꾸지 않고도 사용자 확인값으로 확정할 수 있게 한다.
  - 등록과 상세 수정에서 확인한 계절은 이후 자동 분석보다 우선하고 확인 필요 상태를 해제한다.
- [x] 일괄 사진 등록의 계절 확인 결과 안내
  - 여러 장을 자동 저장한 뒤 계절 확인이 필요한 옷의 개수를 알려주고 보완 필터로 연결한다.
  - 일괄 등록 자체는 중간 실패와 관계없이 기존처럼 완료되게 한다.
- [x] 계절 수정 후 추천 갱신 회귀 검증
  - 사용자가 계절을 수정한 뒤 홈과 코디 추천이 최신 저장값으로 다시 계산되는지 자동 테스트로 고정한다.
  - 날씨 추천과 일반 추천의 기존 입력 통일은 유지한다.

완료된 P0 11차 항목:

- [x] 저장 후 계절 확인 상태의 일관된 표시
  - 계절 후보가 채워져 있어도 AI와 규칙이 충돌해 확인이 필요한 옷은 옷장과 상세에서 놓치지 않게 표시한다.
  - 계절만 확인하면 되는 옷은 일반적인 정보 부족과 구분해 사용자가 바로 수정할 항목을 알 수 있게 한다.
- [x] 옷장의 확인 필요 아이템 모아보기
  - 종류, 색상, 계절을 보완해야 하는 옷만 빠르게 찾을 수 있는 필터를 제공한다.
  - 기존 카테고리와 상세 종류 필터 흐름을 복잡하게 만들지 않는 범위에서 추가한다.
- [x] 계절 보완 편집 동선 단축
  - 계절 확인 배지에서 상세 편집으로 들어왔을 때 사용자가 계절 칩을 바로 찾고 수정할 수 있게 한다.
  - 별도 분석 화면을 만들지 않고 기존 옷 상세 수정 흐름을 재사용한다.

완료된 P0 10차 항목:

- [x] 사진 분석의 계절 판단 근거 강화
  - AI가 의류 종류, 소매, 두께, 보온성, 통기성, 레이어링 역할을 먼저 분석하고 그 근거로 계절을 선택하게 한다.
  - 반팔 니트, 린넨, 패딩, 가디건, 데님 팬츠처럼 자주 틀리는 품목은 서버의 데이터 기반 규칙으로 재검증한다.
- [x] 낮은 신뢰도 계절의 저장 전 확인 강화
  - 계절 근거가 부족한 사진만 사용자 확인 대상으로 표시하고 명확한 분석은 기존처럼 바로 선택한다.
  - AI 내부 점수는 노출하지 않고 계절 칩과 짧은 확인 안내만 사용한다.
- [x] 상품 링크 공식 정보의 계절 보정 연결
  - 상품명과 공식 소재가 있는 링크 등록은 사진 추정보다 신뢰도 높은 계절 근거를 적용한다.
  - 사용자가 직접 수정한 계절은 자동 보정으로 덮어쓰지 않는다.

완료된 P0 9차 항목:

- [x] 저장 코디의 삭제된 아이템 상태 표시
  - 저장 뒤 옷장에서 삭제된 아이템 수를 코디 카드에 명확히 표시하고, 남은 옷만으로 완전한 코디처럼 보이지 않게 한다.
  - 저장 기록은 임의 삭제하지 않아 사용자가 이름과 메모를 확인한 뒤 직접 유지 또는 삭제할 수 있게 한다.
- [x] 저장 코디 기반 추천 제외의 유효 조합 검증
  - 현재 옷장에 모든 아이템이 남아 있는 저장 코디만 새 추천 제외 조합으로 사용한다.
  - 삭제된 아이템이 섞인 오래된 저장 기록이 정상 추천을 막지 않도록 한다.
- [x] 옷 삭제 전 저장 코디 영향 안내
  - 삭제하려는 옷이 저장 코디에 포함되어 있으면 확인창에서 영향을 미리 알려준다.
  - 저장 코디 자체는 유지하고 삭제 후 누락 상태로 표시하는 정책을 일관되게 적용한다.

완료된 P0 8차 항목:

- [x] 프로필 신체 치수 저장 검증
  - 비어 있는 선택 항목은 허용하되 0 이하이거나 숫자가 아닌 치수는 저장 전에 구체적으로 안내한다.
  - 쉼표 소수점을 포함한 유효 치수는 같은 형식으로 정규화하고 실제 유효값만 입력 개수로 센다.
- [x] 잘못 입력한 상품 실측 행 삭제
  - 확정 상품의 사이즈별 실측 목록에서 불필요한 행을 직접 삭제할 수 있게 한다.
  - 마지막 행을 삭제해도 확정 상품 정보는 유지하고 자동 추천만 실측 부족 상태로 돌아가게 한다.
- [x] 오래된 기준 옷 참조 정리
  - 삭제된 옷 ID가 프로필 기준 옷에 남아 있으면 화면과 추천에서 사용하지 않도록 안전하게 정리한다.
  - 정상 기준 옷과 카테고리별 설정은 그대로 유지한다.

완료된 P0 7차 항목:

- [x] 실측 기반 사이즈 추천 핵심 회귀 테스트
  - 하의 길이 우선 추천, FREE 실측 분석, 프로필 부족 차단, 신발 자동 추천 제외를 실제 추천 함수로 검증한다.
  - 프로필 실측이 부족할 때 같은 카테고리의 기준 옷 실측으로 추천을 이어가는 흐름을 검증한다.
- [x] 상품 실측 직접 입력의 숫자 검증 강화
  - 0 이하이거나 숫자가 아닌 실측값이 저장되지 않게 하고, 적어도 한 개의 유효 실측이 있을 때만 행을 저장한다.
  - 같은 사이즈를 다시 입력하면 중복 행을 만들지 않고 기존 행을 안전하게 교체한다.
- [x] 사이즈 추천 차단 상태의 다음 행동 안내 통일
  - 프로필 치수 부족, 상품 실측 부족, 상품 사이즈 미선택 상태를 구분해 사용자가 바로 보완할 수 있게 한다.
  - 자동 추천 대상이 아닌 신발과 액세서리에는 불필요한 차단 카드를 만들지 않는다.

완료된 P0 6차 항목:

- [x] 옷 상세 수정의 기본 정보 검증 연결
  - 상세 수정에서도 종류, 색상, 계절이 빈 값으로 저장되지 않게 등록과 같은 검증 규칙을 사용한다.
  - 기존의 불확실한 값은 사용자가 확인한 뒤 유지할 수 있게 해 수정 흐름을 막지 않는다.
- [x] 사진 없는 옷의 사진 보완 흐름 추가
  - 직접 등록한 옷 상세에서 사진을 선택해 대표 이미지를 보완할 수 있게 한다.
  - 기존 상품 이미지나 배경제거 이미지가 있으면 현재 이미지 우선순위를 유지한다.
- [x] 추천에 쓰기 어려운 옷의 보완 안내
  - 종류나 색상이 불확실해 추천에서 제외되거나 낮게 반영되는 옷을 옷장과 상세에서 구분한다.
  - 사용자가 바로 수정 화면으로 들어가 필요한 정보만 보완하게 한다.

완료된 P0 5차 항목:

- [x] 사진 없는 직접 등록 옷의 표시 안정화
  - 빈 이미지 주소를 이미지 컴포넌트에 전달하지 않고 옷장, 상세, 홈과 추천 카드에서 일관된 대체 화면을 보여준다.
  - 상품 링크와 사진으로 등록한 기존 이미지 우선순위는 유지한다.
- [x] 등록 경로별 필수 데이터 검증 통일
  - 링크, 사진, 직접 등록 모두 추천에 필요한 종류, 색상, 계절이 비어 있는 채 저장되지 않게 한다.
  - 자동 분석값이 불확실하면 저장을 막기보다 사용자가 확인할 항목을 짧게 안내한다.
- [x] 등록 출처와 무관한 추천 입력 정규화
  - 링크, 사진, 직접 등록으로 만든 옷이 같은 카테고리, 계절, 스타일 fallback 규칙을 거쳐 추천에 들어가게 한다.
  - 기존 저장 데이터의 빈 값도 추천 계산 전에 안전하게 정규화한다.

완료된 P0 4차 항목:

- [x] 추천 결과 화면의 점수·등급 노출 제거
  - 추천 계산과 품질 필터에는 점수를 유지하되 사용자 화면에서는 제목, 아이템, 태그와 이유만 보여준다.
  - 저장 코디와 다른 버전에서도 내부 점수나 등급 대신 코디 이름과 구성을 보여준다.
- [x] 코디 분석 결과와 기록의 점수 중심 UI 제거
  - 큰 총점, 세부 점수, 점수순 정렬은 내부 계산에만 남기고 결과 요약과 개선 행동을 중심으로 보여준다.
  - 분석 기록에서도 점수 대신 날짜, 요약, 핵심 개선점을 빠르게 확인하게 한다.
- [x] 상품 실측 자동 추출의 기본 동작과 실패 상태 정리
  - 실측 추출이 환경변수 미설정으로 조용히 꺼지지 않게 기본 정책을 명확히 한다.
  - 지원 상품에서는 자동 추출하고 실패해도 상품 등록은 유지하며 직접 입력으로 연결한다.
- [x] 상품 링크 등록과 이미지 AI 분석 장애 분리
  - 공식 상품 정보를 가져온 뒤 이미지 분석이 실패해도 확인된 상품 정보로 등록을 계속할 수 있게 한다.
  - 불확실한 종류, 색상, 계절은 사용자가 저장 전 또는 상세 화면에서 바로잡게 한다.
- [x] 최소 직접 등록 흐름 추가
  - 링크와 사진을 사용할 수 없을 때 종류, 색상, 계절, 사이즈만 입력해 옷장에 추가할 수 있게 한다.
  - 직접 등록은 상품 링크와 사진 등록을 대체하지 않는 마지막 보조 흐름으로 유지한다.
- [x] 등록부터 추천까지 핵심 흐름 통합 회귀 테스트 추가
  - 상품 링크 추출, 옷장 저장, 상세 수정, 추천 후보 반영의 연결이 끊기지 않는지 자동 검증한다.

완료된 P0 3차 항목:

P0 3차 항목은 아래 순서로 완료했다.

- [x] 상품 링크 등록 실패 흐름 정리
  - 지원하지 않는 쇼핑몰, 상품 이미지 없음, 공식 정보 추출 실패를 구분한다.
  - 실패 이유를 짧게 안내하고 사진 등록 또는 직접 등록으로 바로 이어지게 한다.
- [x] 실측 기반 핏 추천의 신뢰도 기준 정리
  - 상품 실측이나 사용자 신체 치수가 부족할 때는 핏을 추측하지 않는다.
  - 부족한 정보와 사용자가 다음에 할 행동만 명확히 보여준다.
- [x] 오늘의 추천 코디 이유 단순화
  - 날씨, 상황, 실루엣을 반영한 핵심 이유를 사용자 언어로 두세 개만 보여준다.
  - 점수, confidence, 내부 breakdown은 노출하지 않는다.
- [x] 핵심 추천 흐름 회귀 테스트 추가
  - 홈과 추천 화면의 결과 일치, 계절 제외, 품질 기준, 중복 조합, 다른 버전을 자동 검증한다.
- [x] 등록 정보 수정 흐름 정리
  - 잘못 분류된 종류, 계절, 소재, 사이즈를 옷 상세 화면에서 쉽게 바로잡게 한다.
  - 공식 상품 정보, 사진 분석값, 사용자 수정값의 우선순위를 일관되게 유지한다.
- [x] 상품 링크 등록 지원 범위 점검
  - 무신사 외 링크에서 상품명, 이미지, 브랜드, 소재를 어디까지 안정적으로 가져오는지 확인한다.
  - 지원 범위를 벗어난 링크는 성공처럼 보이지 않게 처리한다.

## 완료 기준

- 기존 기능을 삭제하지 않는다.
- 사용자 화면은 결과 중심으로 단순해진다.
- `npm run lint`를 통과한다.
- `npx tsc --noEmit`을 통과한다.
- 로컬 커밋을 만들고 SHA를 보고한다.
- 검증이 모두 통과하면 `origin/main`에 자동 push한다.
- 문제가 있거나 작업 범위가 모호하면 push하지 않고 사용자에게 먼저 묻는다.
